## 基本フィールド（必須）

| フィールド | オプション | 説明 |
|------------|------------|------|
| Status | Backlog/Todo/In progress/Done | 作業状態 |
| 優先度 | P0/P1/P2 | 緊急度 |
| サイズ | XS/S/M/L/XL | 作業規模 |
| 作業種別 | 機能開発/バグ修正/リファクタリング/ドキュメント/テスト/設計 | 作業タイプ |
| 難易度 | 簡単/普通/難しい/要検討 | 技術的難易度 |

| 開始日 | 日付 | 作業開始日 |
| 目標日 | 日付 | 目標完了日 |

## 日付フィールド規則

**開始日設定基準**:
- 状態が `In Progress` に変更される時に開始日を入力
- 開始日 = 実際の作業開始日（Issue作成日ではない）

**目標日設定基準**:
- 作業完了予想日
- サイズ基準で目標日を算定:

| サイズ | 目標日（開始日基準） |
|--------|---------------------|
| XS | 当日 |
| S | +1日 |
| M | +2〜3日 |
| L | +3〜5日 |
| XL | 別途協議（Epic分割推奨） |

**状態変更時の日付更新**:
| 状態変更 | 日付アクション |
|----------|----------------|
| Backlog → Todo | 日付未設定（作業開始前） |
| Todo → In progress | 開始日 = 実際の作業開始日 |
| In progress → Done | 目標日 = 実際の完了日に更新 |

## スクラムフィールド（必須）

| フィールド | タイプ | 説明 | 例 |
|------------|--------|------|-----|
| ストーリーポイント | 数値 | 作業複雑度（フィボナッチ） | 1, 2, 3, 5, 8 |
| スプリント | イテレーション | スプリント割当 | Iteration 1 |
| スプリント目標 | テキスト | このIssueで達成する目標 | "マークダウンパーサー実装" |
| 完了定義 | テキスト | 完了条件（DoD） | "テスト80%, CI通過" |
| ブロッカー | テキスト | 依存関係/障害要素 | "依存: #20" |

## Status ワークフロー

```
Backlog → Todo → In progress → Done
```

| Status | 用途 | 対象 |
|--------|------|------|
| Backlog | 未来の作業キュー | 非活性Epic |
| Todo | 現スプリント作業リスト | 活性EpicのFeat |
| In progress | 現在進行中 | 作業中のEpic + Feat |
| Done | 完了 | マージ/クローズしたIssue |

### Epic vs Feat 状態パターン

| Issueタイプ | Backlog | Todo | In progress | Done |
|-------------|---------|------|-------------|------|
| Epic | 非活性待機 | - | 活性作業中 | 全Feat完了 |
| Feat | - | 作業待ち | 実装中 | PRマージ完了 |
