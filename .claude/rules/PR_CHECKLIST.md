## コードレビュー（レビュー要請時）

**コードレビューを要請するとメイン会話で直接検証を実行します。**

### ドキュメント検証

- [ ] PRタイトル形式: `[タイプ] 説明 (#Issue番号)`
- [ ] Issue連結: `Closes #Issue番号`
- [ ] Mermaidダイアグラム: 最低1つ
- [ ] PR属性: Assignees, Labels

### コードレビュー

**アーキテクチャ**:
- [ ] レイヤー間依存性ルール準拠（逆方向依存 = CRITICAL）
- [ ] インターフェース基盤依存性注入

**コード品質**:
- [ ] エラーラッピング (`fmt.Errorf("...: %w", err)`)
- [ ] エラー無視なし (`_, _ =` なしに)
- [ ] SQL injection防止 (Prepared statement)
- [ ] XSS防止 (HTMLエスケープ)

**スタイル**:
- [ ] 日本語コメント 100%
- [ ] ネーミング規則準拠
- [ ] ハードコーディング値の定数化

**テスト**:
- [ ] コアロジックにテスト存在
- [ ] テストカバレッジ 80%以上
- [ ] テーブルドリブンテストパターン

### 深刻度基準

| 深刻度 | 意味 | 例 |
|--------|------|------|
| CRITICAL | PRブロック | 逆方向依存、SQL injection、Issue連結なし |
| HIGH | 修正推奨 | エラー未処理、Mermaid欠落 |
| MEDIUM | 改善提案 | ハードコーディング、不要な割当 |

## チェックリスト

### 1. コード品質

- [ ] CI通過 (`gofmt`, `golangci-lint`, `go test`)
- [ ] templ生成最新 (`templ generate`)
- [ ] 不要なコード/コメント/デバッグコード削除

### 2. アーキテクチャ検証

- [ ] レイヤー間依存性マトリックス準拠
- [ ] インターフェース基盤DI

### 3. セキュリティ検証

- [ ] SQL injection防止
- [ ] XSS防止（ユーザー入力エスケープ）
- [ ] CSPヘッダー設定

### 4. コードスタイル

- [ ] 日本語コメント
- [ ] ネーミング規則準拠
- [ ] `gofmt` / `goimports` 適用

### 5. ドキュメント更新

- [ ] Mermaidダイアグラム含む

### 6. PR設定

- [ ] PRタイトル: `[タイプ] 説明 (#Issue番号)`
- [ ] Assignees, Labels

### 7. マージ前最終確認

- [ ] CI通過
- [ ] **ユーザー承認確認**（Claude自動マージ禁止）
