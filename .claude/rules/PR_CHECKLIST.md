## 코드 리뷰 (리뷰 요청 시)

**코드 리뷰를 요청하면 메인 대화에서 직접 검증을 수행합니다.**

### 문서 검증

- [ ] PR 제목 형식: `[타입] 설명 (#이슈번호)`
- [ ] 이슈 연결: `Closes #이슈번호`
- [ ] Mermaid 다이어그램: 최소 1개
- [ ] PR 속성: Assignees, Labels

### 코드 리뷰

**아키텍처**:
- [ ] 레이어 간 의존성 규칙 준수 (역방향 의존 = CRITICAL)
- [ ] 인터페이스 기반 의존성 주입

**코드 품질**:
- [ ] 에러 래핑 (`fmt.Errorf("...: %w", err)`)
- [ ] 에러 무시 없음 (`_, _ =` 없이)
- [ ] SQL injection 방지 (Prepared statement)
- [ ] XSS 방지 (HTML 이스케이프)

**스타일**:
- [ ] 일본어 주석 100%
- [ ] 네이밍 규칙 준수
- [ ] 하드코딩 값 상수화

**테스트**:
- [ ] 핵심 로직에 테스트 존재
- [ ] 테스트 커버리지 80% 이상
- [ ] 테이블 드리븐 테스트 패턴

### 심각도 기준

| 심각도 | 의미 | 예시 |
|--------|------|------|
| CRITICAL | PR 차단 | 역방향 의존, SQL injection, 이슈 연결 없음 |
| HIGH | 수정 권장 | 에러 미처리, Mermaid 누락 |
| MEDIUM | 개선 제안 | 하드코딩, 불필요한 할당 |

## 체크리스트

### 1. 코드 품질

- [ ] CI 통과 (`gofmt`, `golangci-lint`, `go test`)
- [ ] templ 생성 최신 (`templ generate`)
- [ ] 불필요한 코드/주석/디버그 코드 제거

### 2. 아키텍처 검증

- [ ] 레이어 간 의존성 매트릭스 준수
- [ ] 인터페이스 기반 DI

### 3. 보안 검증

- [ ] SQL injection 방지
- [ ] XSS 방지 (사용자 입력 이스케이프)
- [ ] CSP 헤더 설정

### 4. 코드 스타일

- [ ] 한국어 주석
- [ ] 네이밍 규칙 준수
- [ ] `gofmt` / `goimports` 적용

### 5. 문서 업데이트

- [ ] Mermaid 다이어그램 포함

### 6. PR 설정

- [ ] PR 제목: `[타입] 설명 (#이슈번호)`
- [ ] Assignees, Labels

### 7. 머지 전 최종 확인

- [ ] CI 통과
- [ ] **사용자 승인 확인** (Claude 자동 머지 금지)
